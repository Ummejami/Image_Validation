# 🧪 Manual QA Testing – Profile Image Upload

## 📌 Project Overview
This project contains manual test scenarios and findings for the **Profile Image Upload** feature.

The goal was to validate functionality, identify edge cases, and uncover potential bugs in the system.

---

## 🎯 Objectives
- Verify image upload functionality
- Validate file type and size restrictions
- Test system behavior under different conditions
- Identify UI/UX inconsistencies
- Detect backend or API-related issues

---

## 🧪 Test Coverage
The following types of testing were performed:

- ✅ Functional Testing  
- ❌ Negative Testing  
- 📏 Boundary Testing  
- 🔄 Edge Case Testing  
- 🎨 UI/UX Testing  
- ⚡ Repetition / Stress Testing  

---

## 📊 Test Summary

| Metric            | Count |
|------------------|------|
| Total Test Cases | 17  |
| Passed           | 6   |
| Failed           | 11   |
| Blocked          | 0    |

> ⚠️ Note: Multiple high-severity issues were identified.

---

## 🐞 Key Bugs Identified

- ❌ Invalid file types (e.g., `.txt`) are accepted  
- ❌ Image upload succeeds but fails during save  
- ❌ Profile image mismatch across different sections  
- ❌ System fails after multiple upload attempts  
- ❌ Upload continues even after page refresh  
- ❌ Persistent error: *"Error saving your avatar, try again"*
